MASTER PROMPT — INVENTORY MODULE
(Automobile Service ERP | Simple for Users | Powerful for System)
ROLE & OBJECTIVE
You are a senior product engineering team designing a perfect Inventory Module for an Automobile Car & Bike Service Management System.
The inventory module must be:
Extremely easy to use for service advisors & storekeepers
Error-proof for daily operations
Fully traceable for management & audit
Deeply integrated with:
Job Cards
Service Operations
Contracts
Accounts
Supplier Module
Zero hard-coding, fully configurable via Admin Panel
1. CORE INVENTORY DESIGN PRINCIPLES (NON-NEGOTIABLE)
Users should never think about stock math
System should guide the user, not confuse them
No part should move without:
Reason
Reference (Job / PO / Transfer)
Timestamp
Inventory must never block service unexpectedly
Every part must be traceable:
Supplier → Store → Job → Invoice
2. INVENTORY MODULE STRUCTURE (USER-FRIENDLY)
The Inventory module must be organized into simple, logical sections:
Item Master
Stock Overview
Job-Based Reservation & Issue
Purchase & Supplier Flow
Returns & Adjustments
Inter-Branch Transfers
Inventory Dashboards & Alerts
3. ITEM MASTER (FOUNDATION – SIMPLE UI)
Item Master Must Capture:
Item ID (auto)
Item name (human-friendly)
Category:
Spare Part
Consumable
Accessory
Tool
OEM / Aftermarket
Brand
Compatible vehicle models
Unit of measure
Reorder level
Max stock level
Tax category
Warranty eligible (Yes / No)
Returnable (Yes / No)
Active / Discontinued
UI Requirement
Search by name / part no / vehicle
Bulk upload supported
No duplicate item creation allowed
4. STOCK OVERVIEW (ZERO CONFUSION)
Stock Must Be Shown As:
Available Stock
Reserved Stock
Issued Stock
User View
Green → Available
Yellow → Reserved
Red → Low / Out of Stock
System Rules
Reserved stock cannot be issued to another job
Stock updates are real-time
Branch-wise visibility mandatory
5. JOB-BASED RESERVATION & ISSUE (MOST IMPORTANT)
At Job Card / Estimate Stage
System must:
Auto-check stock
Auto-reserve required parts
Show shortage clearly
User should see:
“Available”
“Reserved for this job”
“Need to procure”
At Service Execution Stage
Parts can be issued only against Job Card
Issue must capture:
Job Card ID
Technician name
Issue date & time
Partial issue allowed
Excess usage must require reason
⚠️ No manual stock deduction allowed
6. CONTRACT & WARRANTY AWARE INVENTORY
If customer has active contract:
Covered parts auto-marked as “Contract Covered”
Billing disabled for covered parts
Warranty parts:
Tagged for warranty claim
Separate traceability
Contract consumption auto-updated

Scrap Handling

Approval required
